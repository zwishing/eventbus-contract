# EventBus 0.2.0 — Design Spec

**Date:** 2026-05-06
**Status:** Approved scope, awaiting plan
**Crate:** `eventbus-contract` 0.1.0 → workspace `eventbus` 0.2.0

## 1. 目标

按 review 结论修复 P0 + P1 的所有架构与正确性问题，建立可承载未来 Kafka / NATS sibling crate 的 contract crate 形态。0.2.0 是破坏性版本，迁移阵痛一次到位。

## 2. 范围决策

| # | 决策 | 选项 |
|---|------|------|
| 1 | 版本 / 范围 | 0.2.0 破坏性升级，覆盖 review P0 + P1 |
| 2 | Crate 形态 | 一次性拆分为 workspace |
| 3 | 对象安全方案 | dyn-safe 核心 trait + `*Ext` 扩展 trait |
| 4 | Delivery / ack 语义 | 拆分 `Delivery` + `DeliveryControl`，`ack/nack/retry` 消耗 `Box<Self>` |

剩余项按 review 推荐缺省值：newtype、`BatchOutcome`、`SubscriptionConfig` builder、错误链替换、limiter 竞争修复、tracing feature、memory backend 移到 test-utils、AppendRequest 用 `Bytes`、`StreamSubscription` 加 abort + drop observer、PanicObserver 钩子。

## 3. 架构

### 3.1 Workspace 布局

```
eventbus/
├── Cargo.toml                  # [workspace], 共享依赖版本
└── crates/
    ├── eventbus-core/          # traits + Message/PublishOptions/SubscriptionConfig + EventBusError + contract enums + DeliveryState
    ├── eventbus-memory/        # MemoryBackend，feature `test-utils` 默认开启
    ├── eventbus-redis/         # RedisBackend + JsonCodec，features: `tls`
    ├── eventbus-outbox/        # OutboxStore + StateTransitionStore + DeadLetterStore + Dispatcher
    ├── eventbus-integration/   # IntegrationEvent + MessageFactory + EventPublisher
    └── eventbus/               # facade，按 feature 重新导出
```

**facade `eventbus` features:**

- `memory` (default): re-export `eventbus-memory`
- `redis`: re-export `eventbus-redis`
- `outbox`: re-export `eventbus-outbox`
- `integration`: re-export `eventbus-integration`
- `tracing`: 在所有 crate 中开启 `tracing` instrumentation
- `serde` (default): 序列化派生

### 3.2 核心 trait 表面（`eventbus-core`）

```rust
pub type BoxFuture<'a, T> = Pin<Box<dyn Future<Output = T> + Send + 'a>>;
pub type BoxedError = Box<dyn std::error::Error + Send + Sync + 'static>;

pub trait Delivery: Send + Sync {
    fn message(&self) -> &Message;
    fn state(&self) -> &DeliveryState;
}

pub trait DeliveryControl: Send {
    fn ack(self: Box<Self>) -> BoxFuture<'static, Result<(), EventBusError>>;
    fn nack(self: Box<Self>, reason: BoxedError)
        -> BoxFuture<'static, Result<(), EventBusError>>;
    fn retry(self: Box<Self>, reason: BoxedError)
        -> BoxFuture<'static, Result<(), EventBusError>>;
}

pub trait DeliveryHandle: Delivery + DeliveryControl {}
impl<T: Delivery + DeliveryControl> DeliveryHandle for T {}

pub trait Handler: Send + Sync {
    fn handle(&self, delivery: Box<dyn DeliveryHandle>)
        -> BoxFuture<'_, Result<(), EventBusError>>;
}

pub trait Publisher: Send + Sync {
    fn publish(&self, msg: Message, opts: PublishOptions)
        -> BoxFuture<'_, Result<MessageId, EventBusError>>;
    fn publish_batch(&self, msgs: Vec<Message>, opts: PublishOptions)
        -> BoxFuture<'_, Result<BatchOutcome, EventBusError>>;
}

pub trait Subscriber: Send + Sync {
    fn subscribe(&self, cfg: SubscriptionConfig, handler: Arc<dyn Handler>)
        -> BoxFuture<'_, Result<Arc<dyn Subscription>, EventBusError>>;
}

pub trait Subscription: Send + Sync {
    fn name(&self) -> &str;
    fn close(self: Arc<Self>) -> BoxFuture<'static, Result<(), EventBusError>>;
}

pub trait Bus: Publisher + Subscriber {}
impl<T: Publisher + Subscriber + ?Sized> Bus for T {}
```

**扩展 trait（单态化便利层）**

```rust
pub trait PublisherExt: Publisher {
    fn publish_iter<I>(&self, msgs: I, opts: PublishOptions)
        -> BoxFuture<'_, Result<BatchOutcome, EventBusError>>
    where I: IntoIterator<Item = Message> + Send + 'static;
}
impl<P: Publisher + ?Sized> PublisherExt for P { /* 默认实现：collect 到 Vec 后调用 publish_batch */ }

pub trait SubscriberExt: Subscriber {
    fn subscribe_with<H: Handler + 'static>(
        &self, cfg: SubscriptionConfig, handler: H,
    ) -> BoxFuture<'_, Result<Arc<dyn Subscription>, EventBusError>>;
}
```

### 3.3 关键类型

**Newtype（`#[repr(transparent)]`，`Deref<Target = str>`，`From<&str>` 不可用 —— 强制走 `new` 校验）**

```rust
pub struct Topic(String);          // 校验：非空、无控制字符、≤ MAX_TOPIC_LEN
pub struct ConsumerGroup(String);  // 校验：非空
pub struct ConsumerName(String);   // 校验：非空，构造器提供 `auto()` 生成
pub struct MessageId(String);      // 不校验内容，仅类型隔离
```

**`PublishOptions`：保持现有 builder + `#[non_exhaustive]`，无破坏性变化。**

**`SubscriptionConfig` 改造：闭合字段 + builder**

```rust
#[non_exhaustive]
#[derive(Clone, Debug)]
pub struct SubscriptionConfig { /* private */ }

impl SubscriptionConfig {
    pub fn builder(topic: Topic, group: ConsumerGroup) -> SubscriptionConfigBuilder;
    pub fn topic(&self) -> &Topic;
    pub fn consumer_group(&self) -> &ConsumerGroup;
    /* 其他 getter */
}

#[must_use]
pub struct SubscriptionConfigBuilder { /* ... */ }
impl SubscriptionConfigBuilder {
    pub fn consumer_name(self, n: ConsumerName) -> Self;
    pub fn ack_mode(self, m: AckMode) -> Self;
    pub fn ordering(self, o: OrderingMode) -> Self;
    pub fn balance(self, b: ConsumerBalanceMode) -> Self;
    pub fn guarantee(self, g: DeliveryGuarantee) -> Self;
    pub fn max_in_flight(self, n: usize) -> Self;
    pub fn max_pending_acks(self, n: usize) -> Self;
    pub fn max_retry(self, n: usize) -> Self;
    pub fn retry_backoff(self, d: Duration) -> Self;
    pub fn dead_letter_topic(self, t: Topic) -> Self;
    pub fn backpressure(self, p: BackpressurePolicy) -> Self;
    pub fn wildcard_topic(self) -> Self;
    pub fn build(self) -> Result<SubscriptionConfig, EventBusError>;  // normalize_and_validate 等价
}
```

**`BatchOutcome`：publish_batch 返回**

```rust
pub struct BatchOutcome {
    /// 与输入顺序一一对应。无论部分失败还是全部失败，每条都有结果。
    pub results: Vec<Result<MessageId, EventBusError>>,
}
impl BatchOutcome {
    pub fn all_ok(&self) -> bool;
    pub fn ok_count(&self) -> usize;
    pub fn err_count(&self) -> usize;
    pub fn errors(&self) -> impl Iterator<Item = &EventBusError>;
    /// 便捷：若全部成功返回 `Ok(messages)`；任一失败返回包裹原 outcome 的 `Err`
    pub fn into_result(self) -> Result<Vec<MessageId>, BatchError>;
}
```

`publish_batch` 不再因首个错误停止调度后续消息；保留 `publish_batch_parallelism` 限流，但所有消息都会 attempt 一次。

### 3.4 StreamBackend 调整

- `read_new` / `reclaim_idle` 仍返回 `Vec<FetchedEntry>`，签名不变。
- bus 层重构：先 `acquire_owned` permit、再按 `permits.len()` 向 backend 取数据，消除 §1.17 竞争。
- `ClaimedMessage::id: String` 文档化为「backend cursor，不透明字符串，编码契约由 backend 定义」，为未来 Kafka 后端将 `partition:offset` 编码进 string 留出空间。

### 3.5 错误链

替换所有 `EventBusError::Internal(format!("...: {err}"))` 为 `EventBusError::source("...", err)`，至少包括：
- `stream/bus.rs` JoinError、TryAcquireError 路径
- delivery task 失败处
- subscription mutex poison 错误（这条无内层 source，保留 `Internal`）

### 3.6 StreamSubscription

```rust
impl StreamSubscription {
    pub fn name(&self) -> &str;
    pub fn is_running(&self) -> bool;
    pub async fn close(&self) -> Result<(), EventBusError>;  // 现有
    pub async fn abort(&self) -> Result<(), EventBusError>;  // 新增：JoinHandle::abort + await
}
```

`Drop`：保持不阻塞；若 task 还未取走，发送 close 信号且通过 `ErrorObserver::on_error(ErrorScope::Drop, ...)` 记录一次"abandoned without close"事件。

### 3.7 Panic / Drop / Tracing 钩子

```rust
pub trait ErrorObserver: Send + Sync {
    fn on_error(&self, scope: ErrorScope, err: &EventBusError);
    /// 默认空实现，向后兼容
    fn on_panic(&self, scope: ErrorScope, payload: &str) {}
}

pub enum ErrorScope { Read, Reclaim, AckFlush, Drop, HandlerPanic }
```

`tracing` feature on 时：
- `#[instrument(skip(...))]` 在 `publish`、`publish_batch`、`subscribe`、`consume_loop` 每轮、`process_single_message`、`mark_acked`、`publish_dead_letter`。
- malformed entry / poison pill 走 `tracing::warn!`。

### 3.9 Memory backend 移到 test-utils

`eventbus-memory` 默认 `test-utils` feature 开启；生产用户显式 `default-features = false` 时不暴露 `MemoryStreamBackend`，避免误用。

## 4. 数据流

### 4.1 Publish 路径

```
caller → Publisher::publish(msg, opts)
       → validate opts + Topic::validate
       → optional delay
       → prepare_message (apply metadata / idempotency / size cap)
       → backend.publish(topic, msg) → MessageId
```

`publish_batch`：clone `opts.metadata` 一次到栈上，所有消息共享；按 `publish_batch_parallelism` spawn JoinSet；**每条都 attempt** 并把 Result 存入 `BatchOutcome.results[i]`。

### 4.2 Subscribe / Consume 路径

```
caller → Subscriber::subscribe(cfg, Arc<dyn Handler>)
       → cfg builder.build() validates
       → backend.create_group
       → spawn ack_flusher task
       → spawn reclaim task (mpsc<Vec<FetchedEntry>>)
       → spawn consume_loop:
            loop {
                acquire_owned permit(s)  // 修 limiter race
                select! {
                    close_rx
                    reclaim_rx
                    backend.read_new(count = permits.len(), ...)
                }
                for entry in batch:
                    spawn handler.handle(Box::new(StreamDelivery { permit, ack_tx, ... }))
            }
       → on shutdown: drain JoinSet, drop ack_tx, await flusher + reclaim, backend.forget_consumer
       → return Arc<dyn Subscription>
```

### 4.3 Delivery finalize

```
Handler 收到 Box<dyn DeliveryHandle>
  → 读 message() / state()
  → 调用 delivery.ack() / nack(reason) / retry(reason)
       → 这些方法消耗 Box<Self>，permit 在 Box drop 时归还 limiter
       → ack: send AckRequest 到 ack_flusher，等 oneshot
       → nack: publish DLQ → ack
       → retry: 若 attempt < max → republish 到原 topic；否则走 nack
  → Handler 直接 drop Box（未 finalize）→ permit 归还但消息未 ack，会被 reclaim
```

类型系统保证：finalize 后 `Box` 被消耗，Handler 拿不到「已 finalize 的句柄」。删除 `AtomicBool finalized` + `reset_finalize`。

## 5. 错误处理

| 路径 | 失败模式 | 处置 |
|------|----------|------|
| `Topic::new` | 校验失败 | `EventBusError::Validation` |
| `publish` | 单条失败 | `Err(EventBusError)` |
| `publish_batch` | 部分失败 | `Ok(BatchOutcome)` 携带每条 Result |
| `publish_batch` | 全条失败 | 仍返回 `Ok(BatchOutcome)` |
| `publish_batch` | 内部错误（spawn / opts validate） | `Err(EventBusError)` |
| backend transient | 重试 | observer 上报 + 指数退避 |
| handler panic | task | observer.on_panic + JoinSet 报错存 first_delivery_error |
| poison pill | 单条 | observer.on_error + DLQ 发布 + ack |
| ack flusher 关闭 | 单条 | `EventBusError::Internal("ack flusher shut down")` 经 oneshot 返回 |
| limiter race | 防御 | acquire-before-fetch 后不再可达；保留 `Internal` 防御 |

## 8. 非范围（明确不做）

- Kafka / NATS backend：留给 0.3.0 sibling crate。
- backend Cursor 关联类型：保持 `String`，仅文档契约化。
- typestate Delivery：与 dyn-safe 路线冲突，不做。
- `Codec` 进一步抽象（如 schema registry）：超出范围。
- async-trait crate：不引入，全用 `BoxFuture`。
- `trait-variant`：不引入，避免宏依赖。

| 风险 | 缓解 |
